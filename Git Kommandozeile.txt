Git Connection und Clone des Repos:

1.) CMD öffnen -> where ssh (Installationspfad sollte vorhanden sein) ... ssh ist eine Möglichkeit (neben html link oder so) ein github repo zu clonen. man braucht keinen login
2.) RSA-Key erstellen -> ssh-keygen -t rsa -b 4096 --> Generating public/private rsa key pair.
3.) File Eingeben um den Key zu speichern -> Enter (Created directory 'C:\Users\Franky/.ssh'.)
4.) Passwort eingeben -> Enter (Your identification has been saved in C:\Users\Franky/.ssh/id_rsa. blablabla. bildchen)
5.) Kreiert zwei Dateien, private key & public key (pub Datei) 
6.) Navigieren zum Verzeichnis wo die id_rsa abgelegt wurde (per default siehe oben das User VZ) 
7.) Öffnen der Publisher Datei im zB Notepad, alles kopieren
8.) Github, Profilbild anklicken -> Settings -> SSH & GPG Keys
9.) Add New Key (copy paste ins Key Feld) und Namen geben
10.) Github, Repository öffnen (https://github.com/frankkovmir/Benzinapp), auf Code Klicken, SSH Key kopieren
11.) CMD öffnen in einem Verzeichnis der Wahl (hier werden die Daten abgelegt)
12.) git clone git@github.com:frankkovmir/Benzinapp.git
13.) Are you sure you want to continue connecting (yes/no/[fingerprint])? --> Yes
14.) Files sollten jetzt in den Ordner geladen sein

Git eigene Branch zum entwickeln erstellen und in die eigene Branch reinpushen:


1.) CMD öffnen aus dem Verzeichnis wo das Repo reingeklont worden ist (da wo die .git auch liegt)
2.) git branch <gewählter name> (einmalig, erstellt die neue branch)
3.) git checkout <gewählter name> (wechselt in die neue branch)

__________ neue Änderungen mitteilen und pushen _________

4.) git status (zeigt updates im branch die commited werden können)
5.) git add .
6.) git commit -m  "message" (commit message im branch, doppelte anführungszeichen)
7.) git push -u origin <gewähler name>
